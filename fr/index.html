Qu'est-ce que Woc.ts ?

Woc (Well-Organized Code) est un framework pour TypeScript.

Woc est destiné aux développeurs qui construisent leurs applications « lentement et surement ». Woc en effet ne fait aucune magie : pas de langage de templates omnipotent, encore moins d'affreux dirty checking à la manière Angular.js. Élaborer un composant avec Woc prend plus de temps que de saisir quelques mots-clés dans un système de templates. En revanche, une fois le composant réalisé, il est réutilisable.

Actuellement 90 Ko minifié (noyau plus principales extensions).

Sous le capot

Une architecture MVC

  • Le modèle : Le développeur construit son modèle comme il le souhaite, par exemple en l'encapsulant dans un service. Ou bien il peut utiliser un dataset (une sorte de « modèle générique »).
  • La vue est faite de composants. Un composant est un répertoire qui regroupe en particulier du code JavaScript, HTML et CSS. Il peut utiliser d'autres composants et des library. Les composants vont du plus petit (un bouton) au plus grand (tout l'écran). L'encapsulation des composants est une grande force de Woc, elle devrait permettre de toujours développer à petite échelle même dans des projets immenses.
  • Le contrôleur : Le développeur crée des services. Un service est un objet unique accessible depuis tout le projet.

Focus performance

Durant la phase de développement les ressources JavaScript et CSS sont toutes chargées directement depuis leurs emplacements. Cela permet un développement et un déboguage habituel.

Lorsque l'application est prête à être déployée, l'outil « woc-make » tournant sous Node.js la prépare en réduisant le nombre des ressources à charger. Dans l'absolu une application Woc pourrait ne charger que deux ressources en plus du noyau Woc : un fichier CSS minifié et un fichier JSON. Le JSON contient les scripts JS minifiés prêts à être interprétés (en suivant cette approche) ainsi que les templates HTML prêts à être insérés dans le DOM.

L'unité de déploiement est le bundle. Le développeur décide de comment son application doit être chargée en répartissant les divers éléments dans des bundles.

Les forces

  • La réutilisation des composants et services ;
  • La légèreté des applications déployées grâce à la maitrise du nombre de ressources à charger ;
  • Une prise en main rapide : Woc est fait pour que vous utilisiez jQuery ou vos bibliothèques de programmation habituelles ;
  • Pas de dépendance du noyau envers une bibliothèque de programmation externe.
