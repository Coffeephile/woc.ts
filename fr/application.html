Créer une application Woc

Le document HTML d'une application Woc en cours d'élaboration ressemble à ceci : 

<!doctype html>
<html data-woc-preload="ext.w todos.w">
<head>
<meta charset="UTF-8">
<base href="/my-app/">
<script src="woc.min.js"></script>
</head>
<body>
<section data-woc-exec="todos"></section>

L'URL de base doit être mentionnée dans l'élément base ou dans un attribut data-woc-base de l'élément html au choix.

Une application Woc est un répertoire. Si ce répertoire est distinct de celui qui sert de base pour les URLs, alors son URL doit être précisée dans un attribut data-woc-exec de l'élément html.

Une application est composée de un ou plusieurs bundle(s). Dans notre exemple, deux bundles sont préchargés en mode « w » (« work in progress », leur développement n'est pas terminé) : « ext » et « todos ». C'est de cette manière, au niveau du préchargement, que l'on précise au noyau quels bundles doivent être chargés en mode normal (déployés) et quels autres en mode « w ».
