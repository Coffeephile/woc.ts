Déployer avec woc-make

L'utilitaire woc-make, qui tourne sur Node.js, fusionne les éléments des bundles afin de réduire les dépendances à charger.

Pour l'utiliser, veuillez d'abord installer Node.js.

Ensuite l'option -h liste les options disponibles :

$ cd path-to-your-project/
$ node _woctools/woc-make.js -h
Syntax: node _woctools/woc-make.js OPTIONS bundle1 bundle2...
	-h --help
	-i --in-project-path path-to-input-dir (default is current)
	-o --out-project-path path-to-output-dir (default is value from -i)
	-e --default-encoding the default enconding (default utf8)
	-f --include-files "list of extensions" (default is "png gif jpg jpeg")
	-j --minify-js true|false (default is true)
	-c --minify-css true|false (default is true)
	-m --minify-html true|false (default is true)
	-r --remove-destination true|false (default is false)

Par exemple pour préparer les bundles ext.w et todos.w :

$ cd path-to-your-project/
$ node _woctools/woc-make.js -r true ext todos

Cette commande crée ou met à jour des dossiers ext-<version> et todos-<version>. Pour demander au moteur de les utiliser, éditer le fichier todos.html et retirer les suffixes .w dans l'attribut data-woc-preload :

<html data-woc-preload="ext-0.5.w todos-0.5.w">

NB. Lorsque Woc.ts passera en version 1.0, la documentation de référence passera en anglais.
