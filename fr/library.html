Inclure une library

Une library encapsule du code JavaScript, des CSS et d'éventuelles autres ressources (des images…) dans un répertoire suffixé par .l et dont le parent est un bundle.

Elle est le moyen idéal de rendre disponible une classe TypeScript auprès de différents composants et services, ou bien d'inclure des fichiers JavaScript externes : jQuery, jQueryUI, Modernizr, etc.

Une library n'est donc pas forcément écrite en TypeScript. L'encapsulation d'une library externe dans un bundle n'est pas non plus obligatoire. Une fois que le bundle dont elle fait partie est chargé, n'importe quelle partie de l'application peut rendre disponible la library via la méthode requireLib (cela lance l'interprétation du code s'il n'a pas déjà été interprété). Ou bien, si un élément requireLib est présent dans un fichier de configuration, le noyau WOT appelle automatiquement la méthode requireLib.

La configuration se trouve dans un fichier « lib.json ». Voici le contenu du fichier « jQueryUI.l/lib.json » :

{
	"wot": "0.5",
	"requireLib": ["jQuery"],
	"script": ["jquery-ui-1.10.4.custom.min.js", "datepicker-fr.js"],
	"css": "jquery-ui-1.10.4.custom.min.css",
	"name": "jQueryUI"
}

Note commune pour les composants, services et libraries : les dépendances envers les libraries sont mentionnées dans un élément requireLib (optionnel).
