Créer un service

Un service encapsule du code JavaScript et d'éventuelles autres ressources dans un répertoire suffixé par .s et dont le parent est un bundle.

Un service est une unique instance d'une classe TypeScript implémentant woc.Service. Une fois que le bundle dont il fait partie est chargé, l'instance du service devient accessible globalement. L'instance du service est créée au premier appel.

Sa configuration est dans un fichier « serv.json ». Voici le contenu du fichier « ext_Helper.s/serv.json » :

{
	"woc": "0.6",
	"script": ["Helper.js"],
	"name": "ext.Helper"
}

Note commune pour les composants, services et libraries : les dépendances envers les libraries sont mentionnées dans un élément requireLib (optionnel).

Note commune pour les composants et services : l'élément name doit contenir le nom complet de la classe qui implémente woc.Component. Le nom du bundle ne devrait pas être relié à un nom de module TypeScript. La règle est de mentionner le module (en remplaçant les points par des underscores) dans le nom du répertoire du composant ou du service.

Voici un exemple de code TypeScript d'un service :

/// <reference path='../../woc.d.ts' />
module my.module {
	'use strict';
	export class MyService {
		constructor(sc: woc.ServiceContext) {
		}
		// My methods here
	}
}
