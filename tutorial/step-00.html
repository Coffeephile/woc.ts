Step 0 - Bootstrapping

Getting started with TypeScript

How to transpile TypeScript to JavaScript:

  1. Install Node.js;
  2. In command line, install the TypeScript compiler on Node.js, with the command npm given on the official Web site: npm install -g typescript;

Then, in order to learning TypeScript, read the excellent Handbook.

About SASS

The SCSS variant of SASS is used in this tutorial. Please install SASS.

But SASS is not required. You can remove the SASS files (.scss) and directly work on CSS files. When the tutorial gives some SCSS code, just go to the step 14 and copy the matching CSS file.

Get the code

Download the last release of the tutorial, then decompress it.

Make the tutorial ready

From the command line, run:

cd path/to/woc-tuto-N-N-N/
node tuto-install

This command creates fifteen directories step-N: they are the fifteen steps of the tutorial.

Use IDEA (WebStorm, PhpStorm, IntelliJ…)

In this tutorial we recommend WebStorm. Download WebStorm (it includes a one month trial license).

With WebStorm, create a new project on the directory woc-tuto-N-N-N, i.e. the parent directory of the fifteen directories step-N. This means that in a single project by WebStorm, there are sixteen applications: fifteen steps and the first-app.

When opening the first file with a suffix .ts (TypeScript), the IDE suggests to create the appropriate “Watcher”; Accept, then uncheck “Immediate file synchronization”. Note: If you use IntelliJ IDEA, you must install an extra plugin: it is the File watchers plugin that triggers the Typescript transpilation.

Same thing for SCSS files. WebStorm will suggests a Watcher for them, and uncheck “Immediate file synchronization”.

Run a Web server and test the steps

All the steps of this tutorial are in directories step-XX. Your work will be in the directory first-app.

Now we need a Web server.

If you use WebStorm, select a browser in Open in Browser from the context menu of the HTML file step-01/index.html. WebStorm will run its internal Web server for you.

Alternatively, you can use the Web server of your choice, like Apache httpd. Install it and configure it to access to the code of the tutorial by HTTP. Then, test a step with an URL like: http://localhost:8080/path/to/the/tutorial/step-01/.

You can test the steps 01 to 15 just by changing the URL.
