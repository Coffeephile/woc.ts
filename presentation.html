Presentation

What is a component?

  • Until it is deployed, each component is a separated directory;
  • A component embeds HTML, CSS, JavaScript files and other resources like images;
  • Each component is a GUI unit that can be reused in other components.

Other kinds of program unit…

  • A service is a singleton shared by all the application;
  • An initializer is a singleton called just after loading;
  • An external library is a wrapper for JavaScript library, like jQuery;
  • A theme contains CSS files and images.

Each of these program units is a separated directory that can be easily copied and reused in several applications.

Lean and simplicity

  • No unused features, the framework core is really just a loader;
  • No magic, just standard JavaScript features;
  • The build task for deployment is easy and efficient, it doesn’t need Grunt or Gulp.js neither Browserify;
  • Today, the dependencies between services and components can be managed by hand.

Performance

  • The Woc.ts component system doesn’t require any polyfill;
  • Once deployed, JavaScript, HTML and CSS codes are minified and merged in a single JSON file and a single CSS file;
  • Once loaded, JavaScript code is lazily evaluated, following this approach.

Future proof

  • The scope of the Woc.ts core is determined: loading and nothing more, we aims to make the core API stable;
  • So, to keep your work up-to-date, we hope you won’t have to redesign all your application two times a year, but, instead, to rewrite sometime some part of the application when, for example, a new version of your template engine is released;
  • The Woc.ts component mechanism was made with the Web component future standard in mind.

Templating

  • Templating systems are loaded as services. Use Vue.js or Handlebars or any of your choice. The first one brings data binding.

MVC

Woc.ts doesn’t force you to use a MVC model.

However MVC is recommended. With modern templating systems like Vue.js, each component contains a small MVC schema. At the application level, the components are the view, then a global model can be implemented as a service, and the business layer as other services.

Work in JavaScript or CoffeeScript with Woc.ts

TypeScript is just syntactic sugar. The developer can use Woc.ts with JavaScript or CoffeeScript. A TypeScript class is an ES5 function that expects to be used with the new operator. A TypeScript module can be implemented as an ES5 object. Example:

module MyModule {
  'use strict';
  export class MyClass {
    constructor(private arg: string) {
    }
    public method1() {
      this.method2();
    }
    private method2() {
      console.log(this.arg);
    }
  }
}

In JavaScript, you can write:

var MyModule = (function () {
  'use strict';
  function MyClass(arg) {
    this.arg = arg;
  }
  MyClass.prototype.method1() {
    this.method2();
  }
  MyClass.prototype.method2() {
    console.log(this.arg);
  }
  return {
    MyClass: MyClass
  };
})();

Woc.ts on Internet Explorer 8

If IE8 is your target, using Woc.ts is possible but difficult. The core of Woc.ts works roughly on IE8, please fill us the issues when they occur. The dialog polyfill (in the service WocTeam.Dialogs) and EasyRouter (in the service WocTeam.WocEasyRouter) will probably have to be fixed. The template engine Vue.js won’t work (IE9+), you can use Handlebars (no data binding) or manually make the DOM with jQuery in the old school way.

Web Components vs Woc Components

The Woc.ts components and Web Components from HTML 5 are two separate things:

  • Shadow DOM: Developers of Woc.ts components can use shadow DOM;
  • Template element: Developers of Woc.ts components can use it, but the Woc.ts principle to write HTML in dedicated HTML files is handier than the template element;
  • Custom element: Could Woc.ts components be converted to custom elements? This would make the initialization depend of a global context. Perhaps it is a good thing, perhaps not. The question is still open, and there is no emergency since the browsers aren’t ready anyway;
  • HTML imports: The Woc.ts loading system will not need HTML imports.

Notice: a Web component can be embedded in a Woc.ts application, as an external library. Of course, a Woc.ts component can use a Web component.
